# 使用文档

## 1. 项目用途

项目用于通过海康 SDK 和 ISAPI 自动验证设备扬声器、拾音器、补光灯等能力是否正常。

核心扬声器测试流程：

1. 向被测设备发送一段专属测试音
2. 由录音设备录制现场音视频
3. 从录像中提取音频
4. 分析是否存在声音，以及是否匹配当前被测设备的测试音

## 2. 环境要求

- Windows 64 位
- Python 3
- 项目内置 `libs/win64` 下的 HCNetSDK 依赖
- 本机已安装 `ffmpeg`

`ffmpeg` 不再通过命令行参数传入，统一从配置文件读取：

文件：

```text
configs/app_config.json
```

示例：

```json
{
  "ffmpeg_path": "D:\\ffmpeg\\bin\\ffmpeg.exe"
}
```

如果 `ffmpeg` 已经在 PATH 中，也可以写成：

```json
{
  "ffmpeg_path": "ffmpeg"
}
```

## 3. 扬声器测试

基础命令：

```powershell
python .\demo_speaker_test.py `
  --host 10.18.117.22 `
  --port 8000 `
  --username admin `
  --password asdf!234 `
  --recorder-pool-config .\configs\recorder_device_pool.json
```

常用参数：

- `--host`：被测设备 IP
- `--voice-channel`：被测设备语音通道，默认 `2`
- `--recorder-pool-config`：录音设备池配置文件
- `--recorder-pool-wait-seconds`：录音设备池最大等待时间，默认 `300`
- `--send-duration`：测试音时长，默认 `4`
- `--similarity-threshold`：匹配阈值，默认 `0.7`
- `--test-tone-id`：测试音 ID，默认使用 `--host`
- `--audio-compression-types`：遍历的 `audioCompressionType` 列表，默认 `auto`
- `--enable-log`：开启 SDK 日志

如果只想测试指定音频编码：

```powershell
python .\demo_speaker_test.py --audio-compression-types G.711ulaw,G.711alaw,PCM
```

录音设备池配置示例：

```json
{
  "devices": [
    {
      "id": "recorder-a-10.40.230.23",
      "host": "10.40.230.23",
      "port": 8000,
      "username": "admin",
      "password": "asdf!234",
      "channel": 0,
      "voice_channel": 1,
      "max_connections": 5
    }
  ]
}
```

## 5. 输出文件

扬声器测试默认输出到：

```text
recordings/speaker_tests/<device-ip>/
```

包括：

- `speaker_test_log_<ip>_<timestamp>.log`
- `speaker_test_reference_<ip>_<audioCompressionType>_<timestamp>_*.wav`
- `speaker_test_record_<ip>_<audioCompressionType>_<timestamp>.mp4`
- `speaker_test_record_audio_<ip>_<audioCompressionType>_<timestamp>.wav`

拾音测试默认输出到：

```text
recordings/pickup_tests/<device-ip>/
```

## 10. 补光灯测试

```powershell
python .\demo_supplement_light_test.py `
  --host 10.18.117.22 `
  --username admin `
  --password asdf!234
```

默认流程：

1. 查询补光灯能力和当前配置
2. 依次设置不同模式和亮度
3. 抓图
4. 使用 `ffmpeg` 读取中心 `40% x 40%` ROI 灰度均值
5. 判断补光灯是否真正开启，以及亮度是否随档位递增

常用参数：

- `--channel`：图像通道
- `--settle-seconds`：每次设置后等待稳定时间
- `--on-threshold`：开灯亮度差阈值
- `--level-threshold`：档位亮度差阈值

输出目录：

```text
recordings/supplement_light_tests/<device-ip>/
```

## 11. 常见问题

如果没有音频分析结论，优先检查：

- `configs/app_config.json` 中的 `ffmpeg_path` 是否正确
- 日志中是否出现 `analysis dependency ready ffmpeg=...`
- 录音设备是否生成了非空录像文件
- 是否出现 `sound analysis done ...`
- 是否出现 `match analysis done ...`

如果 `has_sound=True` 但 `match=False`：

- 说明录像中有声音，但没有达到当前被测设备测试音的匹配阈值
- 可检查设备距离、环境噪声、扬声器音量、麦克风音量
- 可使用 `--test-tone-id` 复现同一设备的测试音
