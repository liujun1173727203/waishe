# 使用文档

## 1. 项目简介

本项目基于海康 `HCNetSDK` 的 Windows 64 位动态库，使用 Python `ctypes` 封装以下能力：

- SDK 初始化与清理
- 设备登录与登出
- 双向语音对讲
- 语音转发与编码音频发送
- 实时取流录像
- ISAPI 音频输入/输出音量控制
- 复合流录像能力检查与自动切换
- 录像音频分析与参考音频匹配
- 扬声器测试用例编排

## 2. 运行环境

- Windows 64 位
- Python 3
- 项目自带 `libs/win64` 海康 SDK 依赖
- 如果需要分析录像音频，需要本机安装 `ffmpeg`

## 3. 主要文件

- `hikvision_voice.py`：SDK 封装与取流录像
- `hikvision_isapi.py`：ISAPI 能力查询、音量调节、复合流配置
- `video_analysis.py`：录像音频剥离、声音判断、参考音频匹配
- `use_cases/voice_talk_cases.py`：随机音频生成与发送
- `use_cases/speaker_test_cases.py`：扬声器测试完整用例
- `demo_stream_record.py`：普通录像示例
- `demo_composite_stream_record.py`：复合流录像示例
- `demo_random_audio_talk_use_case.py`：随机音频发送示例
- `demo_video_analysis.py`：录像音频分析示例
- `demo_speaker_test.py`：扬声器测试示例

## 4. 普通录像

```powershell
python .\demo_stream_record.py `
  --host 192.168.1.64 `
  --port 8000 `
  --username admin `
  --password 12345 `
  --duration 10
```

默认输出路径：
`recordings/streams/<device-ip>/stream_ch<channel>_YYYYMMDD_HHMMSS.mp4`

## 5. 复合流录像

```powershell
python .\demo_composite_stream_record.py `
  --host 192.168.1.64 `
  --port 8000 `
  --username admin `
  --password 12345 `
  --duration 10
```

默认行为：

1. 登录设备
2. 将音频输入、输出音量调到最大
3. 查询 `/ISAPI/Streaming/channels/<trackStreamID>/capabilities`
4. 判断是否支持复合流录像
5. 如果当前 `Audio.enable` 或 `Audio.enabled` 不是 `true`，自动下发配置
6. 开始录像并保存到设备 IP 对应目录

## 6. 随机音频发送

```powershell
python .\demo_random_audio_talk_use_case.py `
  --host 192.168.1.64 `
  --port 8000 `
  --username admin `
  --password 12345 `
  --duration 3
```

默认输出路径：
`recordings/use_cases/<device-ip>/random_audio_YYYYMMDD_HHMMSS.wav`

## 7. 录像音频分析

```powershell
python .\demo_video_analysis.py `
  --video recordings\streams\192.168.1.64\stream_ch1_20260529_120000.mp4 `
  --reference-audio recordings\use_cases\192.168.1.64\random_audio_20260529_115900.wav
```

分析内容：

1. 从录像文件中剥离音频
2. 判断是否存在有效声音
3. 判断是否包含参考音频

## 8. 扬声器测试用例

```powershell
python .\demo_speaker_test.py `
  --host 192.168.1.64 `
  --port 8000 `
  --username admin `
  --password 12345 `
  --record-duration 10 `
  --send-duration 3 `
  --similarity-threshold 0.8
```

默认流程：

1. 获取设备能力，判断是否支持音频输入
2. 校验 SDK 登录成功
3. 将音频输入、输出音量调到最大
4. 开启对讲
5. 开启设备复合流录像
6. 生成随机模拟音频，并将原始数据保存到设备 IP 对应目录
7. 通过 SDK 向设备发送音频数据
8. 保持录像直到指定时长
9. 关闭录像
10. 分析录像结果，判断是否包含原始音频
11. 当最佳相似度大于等于 `0.8` 时，判定为有效

默认输出目录：
`recordings/speaker_tests/<device-ip>/`

输出文件包括：

- `speaker_test_YYYYMMDD_HHMMSS.mp4`
- `speaker_test_YYYYMMDD_HHMMSS.wav`
- `random_audio_YYYYMMDD_HHMMSS.wav`

## 9. 常用接口

- `start_stream_record()`：启动实时取流录像
- `start_voice_forward()`：启动语音转发，可向设备发送编码音频
- `ensure_audio_input_output_volume_max()`：将音频输入、输出调到最大
- `ensure_composite_stream_recording_enabled()`：确保当前录像流带音频
- `get_audio_input_capability_status()`：检查设备是否支持音频输入
- `analyze_sound_presence()`：判断录像音频是否有声音
- `detect_reference_audio()`：判断录像音频是否包含参考音频
- `run_speaker_test()`：执行扬声器测试完整流程

## 10. 注意事项

- `demo_speaker_test.py` 和 `demo_video_analysis.py` 依赖本机安装 `ffmpeg`
- 扬声器测试默认按设备 IP 分目录保存录像和原始音频
- 如果设备型号的 ISAPI 音频节点不同，可根据真实报文继续补充路径兼容
