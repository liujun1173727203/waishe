# 概要设计

## 1. 设计目标

项目目标是为海康设备语音对讲提供一层轻量、可读、可扩展的 Python 封装，屏蔽原始 SDK 的动态库加载、结构体定义和回调注册细节，让业务代码通过统一接口完成语音对讲能力调用。

## 2. 系统边界

输入：

- 设备连接参数：IP、端口、用户名、密码
- 对讲控制参数：对讲通道、对讲模式、回调开关
- 音频编码参数：编码类型、采样率、码率

输出：

- 登录会话对象
- 对讲会话对象
- 录像会话对象
- 语音转发会话对象
- 设备音频编码参数
- SDK 错误信息

外部依赖：

- 海康 `HCNetSDK.dll`
- `HCNetSDKCom` 组件目录
- `libcrypto-3-x64.dll`
- `libssl-3-x64.dll`

## 3. 模块划分

### 3.1 SDK 封装层

文件：`hikvision_voice.py`

职责：

- 加载动态库
- 配置 SDK 初始化路径
- 绑定 HCNetSDK 函数签名
- 提供 Python 风格的业务接口
- 统一错误处理

### 3.1.1 用例层解耦原则

具体业务场景不直接堆叠到 SDK 封装层，而是放到独立的用例层中，例如：

- `use_cases/voice_talk_cases.py`

该层职责：

- 组合多个 SDK 接口完成具体业务流程
- 处理业务输入输出文件
- 做业务级参数检查、编码适配和节奏控制

### 3.2 对讲会话层

对象：

- `VoiceCall`
- `StreamRecorder`
- `VoiceForwardSession`

职责：

- 管理对讲句柄生命周期
- 管理实时预览和录像句柄生命周期
- 注册和保存回调
- 停止对讲并释放资源

### 3.3 示例调用层

文件：`demo_voice_call.py`

职责：

- 解析命令行参数
- 调用封装接口完成登录、对讲、退出
- 打印关键状态和音频回调信息

### 3.4 用例实现层

对象：

- `VoiceTalkUseCases`

职责：

- 生成随机音频文件
- 根据设备编码类型转换音频数据
- 通过语音转发会话向设备发送音频帧
- 将“文件生成 + 编码 + 发送”的业务过程与 SDK 基础接口解耦

## 4. 核心流程

### 4.1 初始化流程

1. 创建 `HikvisionVoiceSDK`
2. 调用 `initialize()`
3. 加载 `libs/win64` 下的 DLL
4. 配置 SDK 和 OpenSSL 依赖路径
5. 调用 `NET_DVR_Init`

### 4.2 登录流程

1. 组装 `NET_DVR_USER_LOGIN_INFO`
2. 调用 `NET_DVR_Login_V40`
3. 返回 `DeviceSession`

### 4.3 双向对讲流程

1. 根据参数或设备信息确定对讲通道
2. 调用 `NET_DVR_StartVoiceCom_V30`
3. SDK 通过回调返回音频数据
4. 停止时调用 `NET_DVR_StopVoiceCom`

### 4.4 语音转发流程

1. 调用 `NET_DVR_StartVoiceCom_MR_V30`
2. 接收设备编码后音频数据
3. 需要时通过 `NET_DVR_VoiceComSendData` 回传音频
4. 结束时调用 `NET_DVR_StopVoiceCom`

### 4.5 取流录像流程

1. 组装 `NET_DVR_PREVIEWINFO`
2. 调用 `NET_DVR_RealPlay_V40` 建立实时预览句柄
3. 调用 `NET_DVR_SaveRealData` 将实时码流保存到本地文件
4. 停止时依次调用 `NET_DVR_StopSaveRealData` 和 `NET_DVR_StopRealPlay`

默认保存策略：

- 当调用方未显式传入文件路径时，录像文件自动保存到 `recordings/streams/<device-ip>/`

### 4.6 随机音频对讲播放用例流程

1. 登录设备并读取当前语音编码参数
2. 生成 3 秒随机单声道 `wav` 文件
3. 按设备编码类型转换为可发送音频帧
4. 启动 `VoiceForwardSession`
5. 按固定节奏分片发送编码音频数据
6. 设备端播放收到的音频

## 5. 设计特点

- 轻量：仅依赖标准库 `ctypes`
- 贴近 SDK：便于排查海康接口问题
- 可扩展：后续可继续封装录音、转码、保存、设备能力查询等能力
- 统一异常：所有 SDK 调用失败统一抛出 `HikvisionSDKError`
- 能力统一：对讲、语音转发、取流录像均通过会话对象管理生命周期
- 用例解耦：业务场景实现放到 `use_cases` 层，SDK 层保持稳定和通用
