# 概要设计

## 1. 设计目标

项目目标是为海康设备提供一层轻量、可读、可扩展的 Python 封装，屏蔽原始 SDK 的动态库加载、结构体定义、回调注册和 ISAPI 透传细节，让业务代码通过统一接口完成语音对讲、语音转发和录像控制。

## 2. 系统边界

输入：

- 设备连接参数：IP、端口、用户名、密码
- 对讲控制参数：通道、模式、回调开关
- 录像控制参数：预览通道、码流类型、链路类型、输出路径、时长
- 复合流切换参数：`trackStreamID` 对应通道配置

输出：

- 登录会话对象
- 对讲会话对象
- 语音转发会话对象
- 录像会话对象
- 复合流状态对象
- SDK 错误信息

外部依赖：

- 海康 `HCNetSDK.dll`
- `HCNetSDKCom` 组件目录
- `libcrypto-3-x64.dll`
- `libssl-3-x64.dll`

## 3. 模块划分

### 3.1 SDK 封装层

文件：`hikvision_voice.py`

职责：

- 加载动态库
- 配置 SDK 初始化路径
- 绑定 HCNetSDK 函数签名
- 通过 `NET_DVR_STDXMLConfig` 透传 ISAPI 配置
- 提供 Python 风格的业务接口
- 统一错误处理

### 3.2 会话对象层

对象：

- `VoiceCall`
- `VoiceForwardSession`
- `StreamRecorder`

职责：

- 管理句柄生命周期
- 保存回调，避免被 Python 回收
- 停止对讲、转发或录像并释放资源

### 3.3 示例层

文件：

- `demo_voice_call.py`
- `demo_stream_record.py`
- `demo_composite_stream_record.py`

职责：

- 解析命令行参数
- 调用封装接口完成业务流程
- 打印关键状态与输出路径

## 4. 核心流程

### 4.1 初始化流程

1. 创建 `HikvisionVoiceSDK`
2. 调用 `initialize()`
3. 加载 `libs/win64` 下的 DLL
4. 配置 SDK 与 OpenSSL 依赖路径
5. 调用 `NET_DVR_Init`

### 4.2 登录流程

1. 组装 `NET_DVR_USER_LOGIN_INFO`
2. 调用 `NET_DVR_Login_V40`
3. 返回 `DeviceSession`

### 4.3 双向对讲流程

1. 根据参数或设备信息确定对讲通道
2. 调用 `NET_DVR_StartVoiceCom_V30`
3. SDK 通过回调返回音频数据
4. 停止时调用 `NET_DVR_StopVoiceCom`

### 4.4 语音转发流程

1. 调用 `NET_DVR_StartVoiceCom_MR_V30`
2. 接收设备编码后音频数据
3. 需要时通过 `NET_DVR_VoiceComSendData` 回传音频
4. 停止时调用 `NET_DVR_StopVoiceCom`

### 4.5 实时取流录像流程

1. 组装 `NET_DVR_PREVIEWINFO`
2. 调用 `NET_DVR_RealPlay_V40` 建立实时预览句柄
3. 调用 `NET_DVR_SaveRealData` 将实时码流保存到本地文件
4. 停止时依次调用 `NET_DVR_StopSaveRealData` 和 `NET_DVR_StopRealPlay`

默认保存策略：

- 当调用方未显式传入文件路径时，录像文件自动保存到 `recordings/streams/<device-ip>/`

### 4.6 复合流录像流程

1. 根据通道和码流类型换算 `trackStreamID`
2. 调用 `GET /ISAPI/Streaming/channels/<trackStreamID>/capabilities`
3. 判断设备是否支持音频节点配置
4. 调用 `GET /ISAPI/Streaming/channels/<trackStreamID>` 获取当前配置
5. 将 `./Audio/enable` 或 `./Audio/enabled` 设置为 `true`
6. 调用 `PUT /ISAPI/Streaming/channels/<trackStreamID>` 配置回设备
7. 启动实时取流并录像
8. 录像 10 秒后停止

## 5. 设计特点

- 轻量：仅依赖标准库 `ctypes` 和 `xml.etree.ElementTree`
- 贴近 SDK：便于排查海康接口问题
- 能力统一：对讲、语音转发、取流录像均通过会话对象管理生命周期
- 可扩展：后续可以继续封装更多 ISAPI 或业务用例
- 统一异常：所有 SDK 调用失败统一抛出 `HikvisionSDKError`
