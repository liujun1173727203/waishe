# 概要设计

## 1. 设计目标

项目目标是为海康设备提供一层轻量、可读、可扩展的 Python 封装，统一管理 SDK 调用、ISAPI 配置和录像分析能力，并通过用例层将这些基础能力编排成可复用的测试流程。

## 2. 系统分层

### 2.1 SDK 层

文件：`hikvision_voice.py`

职责：

- 加载 HCNetSDK 与依赖库
- 初始化与清理 SDK
- 登录、登出设备
- 开启对讲、语音转发、实时取流录像
- 提供原始 `stdxml_config(...)` 给上层透传 ISAPI

### 2.2 ISAPI 层

文件：`hikvision_isapi.py`

职责：

- 封装能力查询、配置获取、配置下发
- 提供音频输入能力判断接口
- 提供输入/输出音量调节接口
- 提供复合流录像自动切换接口

### 2.3 媒体分析层

文件：`video_analysis.py`

职责：

- 从录像文件中剥离音频
- 判断录像音频中是否存在有效声音
- 判断录像音频中是否包含参考音频

### 2.4 用例编排层

文件：

- `use_cases/voice_talk_cases.py`
- `use_cases/speaker_test_cases.py`

职责：

- 管理随机音频生成与发送
- 串联音量设置、对讲、录像、分析等步骤
- 输出结构化测试结果

### 2.5 示例入口层

文件：

- `demo_stream_record.py`
- `demo_composite_stream_record.py`
- `demo_random_audio_talk_use_case.py`
- `demo_video_analysis.py`
- `demo_speaker_test.py`

职责：

- 解析命令行参数
- 调用封装接口执行测试流程
- 打印关键结果与输出路径

## 3. 核心流程

### 3.1 初始化流程

1. 创建 `HikvisionVoiceSDK`
2. 调用 `initialize()`
3. 加载 `libs/win64` 下的 SDK 和 OpenSSL 依赖
4. 设置 SDK 本地初始化路径
5. 调用 `NET_DVR_Init`

### 3.2 登录流程

1. 组装 `NET_DVR_USER_LOGIN_INFO`
2. 调用 `NET_DVR_Login_V40`
3. 返回 `DeviceSession`

### 3.3 普通录像流程

1. 确定预览通道
2. 调用 `start_stream_record(...)`
3. 通过 `NET_DVR_RealPlay_V40` 建立实时流
4. 通过 `NET_DVR_SaveRealData` 保存录像
5. 到达时长后停止保存与预览

### 3.4 复合流录像流程

1. 查询并设置音频输入/输出音量到最大
2. 计算 `trackStreamID`
3. 查询复合流能力
4. 获取当前流配置
5. 检查 `Audio.enable` 或 `Audio.enabled`
6. 若未开启音频，则下发更新后的 ISAPI 配置
7. 启动录像

### 3.5 录像分析流程

1. 使用 `ffmpeg` 从录像中提取音频
2. 转换为统一的 `8000Hz / mono / pcm_s16le wav`
3. 计算分帧 RMS
4. 输出声音存在性结果
5. 如有参考音频，则继续做滑窗相似度匹配

## 4. 用例流程

### 4.1 用例一：普通录像

目标：
对指定设备进行基础实时录像。

具体步骤：

1. 初始化 SDK
2. 登录设备
3. 选择录像通道
4. 启动录像
5. 将录像保存到 `recordings/streams/<device-ip>/`
6. 达到指定时长后关闭录像
7. 登出并清理 SDK

### 4.2 用例二：复合流录像

目标：
确保录像文件中包含设备音频。

具体步骤：

1. 初始化 SDK
2. 登录设备
3. 将音频输入、输出音量调到最大
4. 查询复合流能力
5. 判断设备是否支持复合流录像
6. 如果当前不是复合流音频开启状态，则自动下发配置
7. 开始录像
8. 持续录像指定时长
9. 关闭录像并释放资源

### 4.3 用例三：随机音频发送

目标：
生成可区分的参考音频，并通过对讲发送到设备。

具体步骤：

1. 初始化 SDK
2. 登录设备
3. 查询设备当前音频编码方式
4. 生成随机 PCM 音频
5. 按设备编码格式转成可发送数据
6. 保存参考音频文件
7. 通过 `NET_DVR_StartVoiceCom_MR_V30` 建立语音转发
8. 分帧发送音频数据
9. 关闭对讲并清理资源

### 4.4 用例四：录像结果声音判断

目标：
判断录制结果中是否存在有效声音。

具体步骤：

1. 提取录像音频
2. 统一音频格式
3. 对音频分帧
4. 计算每帧 RMS
5. 用阈值判断是否存在有效声音

### 4.5 用例五：录像结果参考音频包含判断

目标：
判断录像中是否包含预先发送的参考音频。

具体步骤：

1. 准备录像文件
2. 准备参考音频文件
3. 提取录像音频
4. 计算录像音频 RMS 序列
5. 计算参考音频 RMS 序列
6. 对两个序列做去均值和归一化
7. 通过滑窗匹配寻找最佳相似度
8. 根据阈值输出是否匹配

### 4.6 用例六：扬声器测试

目标：
验证设备扬声器链路是否能正确播放并被复合流录像采集到。

前置条件：

- 固定存在一台支持复合流录像的设备 A
- 设备可正常 SDK 登录
- 本机安装 `ffmpeg`

具体步骤：

1. 初始化 SDK
2. 登录设备 A
3. 获取设备能力，判断是否支持音频输入
4. 将音频输入、输出音量调到最大
5. 检查并切换设备到复合流录像状态
6. 开启对讲
7. 开启设备 A 录像
8. 生成可与环境音区分的随机模拟音频
9. 将原始音频保存到设备 IP 对应目录
10. 通过 SDK 向设备发送音频
11. 保持录像直到指定时长
12. 关闭录像与对讲
13. 从录像中剥离音频
14. 判断录像中是否有声音
15. 判断录像音频中是否包含原始音频
16. 当最佳相似度大于等于 `0.8` 时，判定扬声器测试通过

## 5. 设计特点

- 分层清晰：SDK、ISAPI、用例、分析相互解耦
- 结果结构化：关键流程都返回 dataclass 结果对象
- 可维护：同一能力可在不同用例中复用
- 可扩展：后续可继续增加更多设备验证用例
