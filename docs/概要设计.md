# 概要设计

## 1. 设计目标

项目目标是为海康设备语音对讲提供一层轻量、可读、可扩展的 Python 封装，屏蔽原始 SDK 的动态库加载、结构体定义和回调注册细节，让业务代码通过统一接口完成语音对讲能力调用。

## 2. 系统边界

输入：

- 设备连接参数：IP、端口、用户名、密码
- 对讲控制参数：对讲通道、对讲模式、回调开关
- 音频编码参数：编码类型、采样率、码率

输出：

- 登录会话对象
- 对讲会话对象
- 语音转发会话对象
- 设备音频编码参数
- SDK 错误信息

外部依赖：

- 海康 `HCNetSDK.dll`
- `HCNetSDKCom` 组件目录
- `libcrypto-3-x64.dll`
- `libssl-3-x64.dll`

## 3. 模块划分

### 3.1 SDK 封装层

文件：`hikvision_voice.py`

职责：

- 加载动态库
- 配置 SDK 初始化路径
- 绑定 HCNetSDK 函数签名
- 提供 Python 风格的业务接口
- 统一错误处理

### 3.2 对讲会话层

对象：

- `VoiceCall`
- `VoiceForwardSession`

职责：

- 管理对讲句柄生命周期
- 注册和保存回调
- 停止对讲并释放资源

### 3.3 示例调用层

文件：`demo_voice_call.py`

职责：

- 解析命令行参数
- 调用封装接口完成登录、对讲、退出
- 打印关键状态和音频回调信息

## 4. 核心流程

### 4.1 初始化流程

1. 创建 `HikvisionVoiceSDK`
2. 调用 `initialize()`
3. 加载 `libs/win64` 下的 DLL
4. 配置 SDK 和 OpenSSL 依赖路径
5. 调用 `NET_DVR_Init`

### 4.2 登录流程

1. 组装 `NET_DVR_USER_LOGIN_INFO`
2. 调用 `NET_DVR_Login_V40`
3. 返回 `DeviceSession`

### 4.3 双向对讲流程

1. 根据参数或设备信息确定对讲通道
2. 调用 `NET_DVR_StartVoiceCom_V30`
3. SDK 通过回调返回音频数据
4. 停止时调用 `NET_DVR_StopVoiceCom`

### 4.4 语音转发流程

1. 调用 `NET_DVR_StartVoiceCom_MR_V30`
2. 接收设备编码后音频数据
3. 需要时通过 `NET_DVR_VoiceComSendData` 回传音频
4. 结束时调用 `NET_DVR_StopVoiceCom`

## 5. 设计特点

- 轻量：仅依赖标准库 `ctypes`
- 贴近 SDK：便于排查海康接口问题
- 可扩展：后续可继续封装录音、转码、保存、设备能力查询等能力
- 统一异常：所有 SDK 调用失败统一抛出 `HikvisionSDKError`
