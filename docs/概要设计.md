# 概要设计

## 1. 设计目标

项目目标是为海康设备提供一层轻量、可读、可扩展的 Python 封装，屏蔽底层 SDK 动态库加载、结构体定义、回调注册和 ISAPI 透传细节，让业务代码通过统一接口完成设备控制；同时将录像结果分析独立在设备控制层之外，避免业务流程和媒体分析逻辑相互耦合。

## 2. 系统边界

输入：

- 设备连接参数：IP、端口、用户名、密码
- 对讲控制参数：通道、模式、回调开关
- 录像控制参数：预览通道、码流类型、链路类型、输出路径、时长
- 音量控制参数：输入通道、输出通道
- 复合流切换参数：`trackStreamID` 对应通道配置
- 分析参数：视频文件路径、参考随机音频文件路径、阈值

输出：

- 登录会话对象
- 对讲会话对象
- 语音转发会话对象
- 录像会话对象
- 复合流状态对象
- 音量状态对象
- 视频分析结果对象
- SDK / ISAPI / 视频分析错误信息

## 3. 模块划分

### 3.1 SDK 封装层

文件：`hikvision_voice.py`

职责：

- 加载动态库
- 配置 SDK 初始化路径
- 绑定 HCNetSDK 函数签名
- 提供原始 `stdxml_config(...)`
- 提供 Python 风格的设备控制接口
- 统一 SDK 错误处理

### 3.2 ISAPI 封装层

文件：`hikvision_isapi.py`

职责：

- 基于 `HikvisionVoiceSDK.stdxml_config(...)` 封装 ISAPI 请求
- 提供音量获取与下发能力
- 提供流能力查询、配置获取与下发能力
- 提供复合流录像前的自动切换逻辑

### 3.3 会话对象层

对象：

- `VoiceCall`
- `VoiceForwardSession`
- `StreamRecorder`

职责：

- 管理句柄生命周期
- 保存回调，避免被 Python 回收
- 停止对讲、转发或录像并释放资源

### 3.4 视频分析层

文件：`video_analysis.py`

职责：

- 通过 `ffmpeg` 从录像文件中剥离音频
- 判断录制音频中是否存在有效声音
- 判断录制音频中是否包含参考随机音频

### 3.5 示例层

文件：

- `demo_voice_call.py`
- `demo_stream_record.py`
- `demo_composite_stream_record.py`
- `demo_video_analysis.py`

职责：

- 解析命令行参数
- 调用封装接口完成业务流程
- 打印关键状态、输出路径和分析结果

## 4. 核心流程

### 4.1 初始化流程

1. 创建 `HikvisionVoiceSDK`
2. 调用 `initialize()`
3. 加载 `libs/win64` 下的 DLL
4. 配置 SDK 与 OpenSSL 依赖路径
5. 调用 `NET_DVR_Init`

### 4.2 登录流程

1. 组装 `NET_DVR_USER_LOGIN_INFO`
2. 调用 `NET_DVR_Login_V40`
3. 返回 `DeviceSession`

### 4.3 普通录像流程

1. 组装 `NET_DVR_PREVIEWINFO`
2. 调用 `NET_DVR_RealPlay_V40`
3. 调用 `NET_DVR_SaveRealData` 保存实时码流
4. 停止时依次调用 `NET_DVR_StopSaveRealData` 和 `NET_DVR_StopRealPlay`

### 4.4 复合流录像流程

1. 调用音量控制接口，将语音输入和输出调到最大
2. 根据通道和码流类型换算 `trackStreamID`
3. 查询复合流能力
4. 获取当前流配置
5. 将 `./Audio/enable` 或 `./Audio/enabled` 设置为 `true`
6. 下发配置回设备
7. 启动实时取流并录像
8. 录像 10 秒后停止

### 4.5 视频分析流程

1. 用 `ffmpeg` 从视频中剥离音频
2. 将音频转成统一格式的 `wav`
3. 按固定帧长计算 RMS
4. 判断是否有有效声音
5. 若提供参考随机音频，再计算帧能量签名并做滑窗相似度匹配

## 5. 用例流程

### 5.1 用例一：普通录像

目标：

- 对指定设备通道进行普通实时录像

具体步骤：

1. 初始化 SDK
2. 登录设备
3. 确定录像通道
4. 调用 `start_stream_record(...)`
5. 将录像文件保存到 `recordings/streams/<device-ip>/`
6. 到达指定时长后停止录像
7. 登出并清理 SDK

### 5.2 用例二：复合流录像

目标：

- 录制包含音频的复合流录像文件

具体步骤：

1. 初始化 SDK
2. 登录设备
3. 调用 `ensure_audio_input_output_volume_max(...)`
4. 调用 `ensure_composite_stream_recording_enabled(...)`
5. 判断设备是否支持复合流录像
6. 若当前不是复合流，则自动下发 ISAPI 配置
7. 启动录像并持续 10 秒
8. 停止录像
9. 登出并清理 SDK

### 5.3 用例三：随机音频对讲验证

目标：

- 向设备发送一段随机生成的音频，用于后续录像验证

具体步骤：

1. 初始化 SDK
2. 登录设备
3. 查询设备当前音频编码方式
4. 生成随机 PCM 音频
5. 按设备编码方式转换音频数据
6. 通过语音转发接口分帧发送到设备
7. 保存本地参考随机音频文件
8. 登出并清理 SDK

### 5.4 用例四：录像结果声音判断

目标：

- 判断录制下来的视频文件中是否存在有效声音

具体步骤：

1. 使用 `ffmpeg` 从视频中剥离音频
2. 统一转成 `8000Hz`、单声道、`pcm_s16le` 的 `wav`
3. 对音频按固定帧长切帧
4. 逐帧计算 RMS
5. 用阈值判断是否存在有效声音

### 5.5 用例五：录像结果随机音频包含判断

目标：

- 判断录制数据中是否包含先前随机生成并发送给设备的音频

具体步骤：

1. 准备录制视频文件
2. 准备随机音频参考文件
3. 从视频中剥离音频
4. 计算录制音频帧 RMS 序列
5. 计算参考随机音频帧 RMS 序列
6. 对两个序列做归一化
7. 通过滑窗匹配寻找最佳相似度
8. 根据阈值判断是否包含参考随机音频

## 6. 设计特点

- 轻量：仅依赖标准库、系统 `ffmpeg` 和海康 SDK
- 分层清晰：SDK 控制、ISAPI 配置和视频分析彼此解耦
- 能力统一：对讲、转发、录像均通过统一会话对象管理
- 可扩展：后续可以继续封装更多 ISAPI 或分析能力
- 统一异常：不同层级都提供独立但一致的异常语义
