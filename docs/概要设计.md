# 概要设计

## 1. 设计目标

项目目标是为海康设备提供一层轻量、可读、可扩展的 Python 封装，屏蔽原始 SDK 的动态库加载、结构体定义、回调注册和 ISAPI 透传细节，让业务代码通过统一接口完成设备控制，并把录制结果分析独立在设备控制层之外。

## 2. 系统边界

输入：

- 设备连接参数：IP、端口、用户名、密码
- 对讲控制参数：通道、模式、回调开关
- 录像控制参数：预览通道、码流类型、链路类型、输出路径、时长
- 音量控制参数：输入通道、输出通道
- 复合流切换参数：`trackStreamID` 对应通道配置
- 分析参数：视频文件路径、参考随机音频文件路径、阈值

输出：

- 登录会话对象
- 对讲会话对象
- 语音转发会话对象
- 录像会话对象
- 复合流状态对象
- 音量状态对象
- 视频分析结果对象
- SDK / ISAPI / 视频分析错误信息

## 3. 模块划分

### 3.1 SDK 封装层

文件：`hikvision_voice.py`

职责：

- 加载动态库
- 配置 SDK 初始化路径
- 绑定 HCNetSDK 函数签名
- 提供原始 `stdxml_config(...)` 能力
- 提供 Python 风格的设备控制接口
- 统一 SDK 错误处理

### 3.2 ISAPI 封装层

文件：`hikvision_isapi.py`

职责：

- 基于 `HikvisionVoiceSDK.stdxml_config(...)` 封装 ISAPI 请求
- 提供音量获取与下发能力
- 提供流能力查询、配置获取与下发能力
- 提供复合流录像前的自动切换逻辑

### 3.3 会话对象层

对象：

- `VoiceCall`
- `VoiceForwardSession`
- `StreamRecorder`

职责：

- 管理句柄生命周期
- 保存回调，避免被 Python 回收
- 停止对讲、转发或录像并释放资源

### 3.4 视频分析层

文件：`video_analysis.py`

职责：

- 通过 `ffmpeg` 从录像文件中剥离音频
- 判断录制音频中是否存在有效声音
- 判断录制音频中是否包含参考随机音频

### 3.5 示例层

文件：

- `demo_voice_call.py`
- `demo_stream_record.py`
- `demo_composite_stream_record.py`
- `demo_video_analysis.py`

职责：

- 解析命令行参数
- 调用封装接口完成业务流程
- 打印关键状态、输出路径和分析结果

## 4. 核心流程

### 4.1 初始化流程

1. 创建 `HikvisionVoiceSDK`
2. 调用 `initialize()`
3. 加载 `libs/win64` 下的 DLL
4. 配置 SDK 与 OpenSSL 依赖路径
5. 调用 `NET_DVR_Init`

### 4.2 登录流程

1. 组装 `NET_DVR_USER_LOGIN_INFO`
2. 调用 `NET_DVR_Login_V40`
3. 返回 `DeviceSession`

### 4.3 普通录像流程

1. 组装 `NET_DVR_PREVIEWINFO`
2. 调用 `NET_DVR_RealPlay_V40`
3. 调用 `NET_DVR_SaveRealData` 保存实时码流
4. 停止时依次调用 `NET_DVR_StopSaveRealData` 和 `NET_DVR_StopRealPlay`

### 4.4 复合流录像流程

1. 调用音量控制接口，将语音输入和输出调到最大
2. 根据通道和码流类型换算 `trackStreamID`
3. 查询复合流能力
4. 获取当前流配置
5. 将 `./Audio/enable` 或 `./Audio/enabled` 设置为 `true`
6. 下发配置回设备
7. 启动实时取流并录像
8. 录像 10 秒后停止

### 4.5 视频分析流程

1. 用 `ffmpeg` 从视频中剥离音频
2. 将音频转成统一格式的 `wav`
3. 按固定帧长计算 RMS
4. 判断是否有有效声音
5. 若提供参考随机音频，再计算帧能量签名并做滑窗相似度匹配

## 5. 设计特点

- 轻量：仅依赖标准库、系统 `ffmpeg` 和海康 SDK
- 分层清晰：SDK 控制、ISAPI 配置和视频分析彼此解耦
- 能力统一：对讲、转发、录像均通过统一会话对象管理
- 可扩展：后续可以继续封装更多 ISAPI 或分析能力
- 统一异常：不同层级都提供独立但一致的异常语义
