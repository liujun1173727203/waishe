# 详细设计

## 1. 文件结构

### 1.1 `hikvision_voice.py`

核心封装文件，包含以下内容：

- 常量定义
- HCNetSDK 结构体定义
- 语音回调函数签名定义
- 数据类定义
- `HikvisionVoiceSDK` 主封装类
- `VoiceCall` 对讲会话类
- `VoiceForwardSession` 语音转发会话类

### 1.2 `demo_voice_call.py`

提供最小可运行示例，验证：

- SDK 初始化
- 对讲模式设置
- 设备登录
- 双向语音对讲
- 回调输出

### 1.3 `demo_stream_record.py`

提供最小可运行示例，验证：

- SDK 初始化
- 设备登录
- 实时取流
- 保存录像文件
- 停止录像并释放资源

## 3. 结构体设计

### 3.1 本地路径结构

`NET_DVR_LOCAL_SDK_PATH`

用途：

- 给 SDK 提供组件目录路径

### 3.2 对讲模式结构

`NET_DVR_LOCAL_TALK_MODE_CFG`

用途：

- 切换库模式对讲或 Windows API 模式对讲

### 3.3 音频压缩结构

`NET_DVR_COMPRESSION_AUDIO`

用途：

- 获取和设置设备当前语音编码参数

### 3.4 实时预览结构

`NET_DVR_PREVIEWINFO`

用途：

- 向 `NET_DVR_RealPlay_V40` 传递通道号、码流类型、链接方式等参数

### 3.5 登录结构

`NET_DVR_USER_LOGIN_INFO`

用途：

- 向 `NET_DVR_Login_V40` 传递登录参数

### 3.6 设备信息结构

- `NET_DVR_DEVICEINFO_V30`
- `NET_DVR_DEVICEINFO_V40`

用途：

- 接收设备返回能力信息
- 提供默认对讲通道等基础数据

## 4. 数据类设计

### 4.1 `AudioCompressInfo`

字段：

- `encode_type`
- `sampling_rate`
- `bit_rate`
- `support_flag`

用途：

- 用 Python 数据对象表达设备音频编码配置

### 4.2 `DeviceSession`

字段：

- `user_id`
- `host`
- `device_info`
- `device_info_v40`

属性：

- `default_voice_channel`
- `default_preview_channel`

用途：

- 表达登录后的设备会话
- 保存后续对讲、配置查询所需的用户句柄和设备信息

## 5. `HikvisionVoiceSDK` 设计

### 5.1 初始化方法

`initialize(enable_log=False, log_dir=None)`

处理内容：

- 加载 SDK
- 配置 SDK 初始化路径
- 绑定接口签名
- 调用 SDK 初始化
- 按需开启日志
- 注册进程退出清理动作

### 5.2 清理方法

`cleanup()`

处理内容：

- 清空回调缓存
- 调用 `NET_DVR_Cleanup`

### 5.3 登录与登出

- `login(host, port, username, password)`
- `logout(session)`

设计说明：

- 登录成功后返回 `DeviceSession`
- 登录失败统一抛出 `HikvisionSDKError`

### 5.4 对讲模式设置

`set_talk_mode(use_windows_api=False)`

设计说明：

- `False` 使用库模式
- `True` 使用旧版 Windows API 模式

### 5.5 音频编码配置

- `get_current_audio_compress(session)`
- `set_audio_compress(session, encode_type, sampling_rate=0, bit_rate=0)`

设计说明：

- 通过 SDK 查询和修改设备语音编码参数

### 5.6 双向对讲

`start_call(session, voice_channel=None, need_pcm_callback=False, audio_callback=None)`

设计说明：

- 创建 `VoiceCall`
- 自动使用设备默认对讲通道
- 支持可选音频回调

### 5.7 语音转发

`start_voice_forward(session, voice_channel=None, encoded_audio_callback=None)`

设计说明：

- 创建 `VoiceForwardSession`
- 回调返回设备编码后的音频数据

### 5.8 取流录像

`start_stream_record(session, file_path, channel=None, stream_type=0, link_mode=0, blocked=True, real_data_callback=None)`

设计说明：

- 创建 `StreamRecorder`
- 默认使用设备默认预览通道
- 默认使用主码流和 TCP 取流
- 通过 `NET_DVR_SaveRealData` 将码流直接保存到本地文件
- 可选透传实时码流回调给上层业务

## 6. 会话对象设计

### 6.1 `VoiceCall`

职责：

- 启动和停止双向对讲
- 管理 `NET_DVR_StartVoiceCom_V30` 返回的句柄
- 保存回调，防止被 Python 垃圾回收

关键方法：

- `start()`
- `stop()`
- `__enter__()`
- `__exit__()`

### 6.2 `VoiceForwardSession`

职责：

- 启动和停止编码音频转发
- 提供发送编码后音频能力

关键方法：

- `start()`
- `send_encoded_audio(data)`
- `stop()`
- `__enter__()`
- `__exit__()`

### 6.3 `StreamRecorder`

职责：

- 启动和停止实时预览
- 启动和停止录像文件保存
- 保存实时码流回调，防止被 Python 垃圾回收

关键方法：

- `start()`
- `stop()`
- `__enter__()`
- `__exit__()`

## 7. 回调设计

回调类型：

- `VOICE_DATA_CALLBACK`
- `REAL_DATA_CALLBACK`

适配逻辑：

- SDK 回调原始指针和长度
- Python 端通过 `ctypes.string_at()` 转换为 `bytes`
- 再交给上层 Python 回调处理

缓存策略：

- 所有活跃回调保存在 `_active_callbacks`
- 对讲停止后从缓存移除

## 8. 错误处理设计

统一异常类型：

- `HikvisionSDKError`

包含信息：

- 业务错误描述
- SDK 接口名
- 错误码
- SDK 错误文本

错误信息来源：

- `NET_DVR_GetLastError`
- `NET_DVR_GetErrorMsg`

## 9. 已知限制

- 当前项目仅面向 Windows 64 位环境
- 当前示例未包含自动化测试
- `README.md` 当前存在历史编码问题，后续如需重写建议统一转为 UTF-8
- 音频数据如何保存、转码、复用，当前仍由上层业务回调自行决定
- 录像文件保存格式取决于设备码流和 SDK 输出，扩展名建议由调用方按实际场景控制

## 10. 变更维护要求

后续每次代码变更，默认同步执行以下动作：

1. 更新实现代码

3. 做最基本验证，例如 `python -m py_compile`
4. 使用详细 commit message 记录修改目的、行为变化、影响范围
5. 提交到当前分支
