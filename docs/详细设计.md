# 详细设计

## 1. 文件结构

### 1.1 `hikvision_voice.py`

核心 SDK 封装文件，包含：

- 常量定义
- HCNetSDK 结构体定义
- 语音和实时码流回调定义
- 数据类定义
- `HikvisionVoiceSDK`
- `VoiceCall`、`VoiceForwardSession`、`StreamRecorder`

### 1.2 `hikvision_isapi.py`

ISAPI 能力封装文件，包含：

- XML 辅助函数
- `CompositeStreamStatus`
- `AudioVolumeStatus`
- `AudioIoVolumeStatus`
- `HikvisionIsapiClient`

### 1.3 `video_analysis.py`

独立的视频分析模块，包含：

- `VideoAnalysisError`
- `SoundAnalysisResult`
- `ReferenceAudioMatchResult`
- `RecordedVideoAnalyzer`

### 1.4 示例脚本

- `demo_voice_call.py`
- `demo_stream_record.py`
- `demo_composite_stream_record.py`
- `demo_video_analysis.py`

## 2. SDK 层设计

### 2.1 `HikvisionVoiceSDK`

主要职责：

- 加载 HCNetSDK
- 初始化 SDK
- 登录 / 登出设备
- 启动对讲 / 语音转发 / 实时录像
- 提供原始 `stdxml_config(...)`

### 2.2 `DeviceSession`

字段：

- `user_id`
- `host`
- `device_info`
- `device_info_v40`

属性：

- `default_voice_channel`
- `default_preview_channel`

### 2.3 `StreamRecorder`

职责：

- 启动实时预览
- 开始保存录像
- 停止保存录像
- 释放句柄和回调

## 3. ISAPI 层设计

### 3.1 `CompositeStreamStatus`

字段：

- `supported`
- `track_stream_id`
- `audio_enabled`
- `changed`

用途：

- 表达复合流能力检查和切换结果

### 3.2 `AudioVolumeStatus`

字段：

- `supported`
- `changed`
- `value`
- `maximum`
- `request_path`

用途：

- 表达单个音量接口的调节结果

### 3.3 `AudioIoVolumeStatus`

字段：

- `input_status`
- `output_status`

用途：

- 表达语音输入和输出整体调节结果

### 3.4 `HikvisionIsapiClient`

主要接口：

- `get_streaming_channel_capabilities(...)`
- `get_streaming_channel_config(...)`
- `set_streaming_channel_config(...)`
- `ensure_composite_stream_recording_enabled(...)`
- `set_audio_input_volume_to_max(...)`
- `set_audio_output_volume_to_max(...)`
- `ensure_audio_input_output_volume_max(...)`

设计说明：

- 基于 `HikvisionVoiceSDK.stdxml_config(...)`
- 通过 `channel + stream_type` 计算 `trackStreamID`
- 音量控制接口与录像逻辑分开封装，降低耦合度
- 复合流录像示例只是在流程中组合调用这些独立接口

### 3.5 音量控制策略

实现要点：

- 输入和输出使用独立接口
- 支持候选配置路径和能力路径回退
- 优先从能力节点上的 `max` 获取最大音量
- 若能力路径不可用，则尝试从配置节点属性中推断最大值
- 目前默认最大音量兜底为 `15`

## 4. 视频分析层设计

### 4.1 `RecordedVideoAnalyzer`

主要接口：

- `extract_audio(video_path, output_path=None)`
- `analyze_sound_presence(video_path, extracted_audio_path=None, rms_threshold=300.0)`
- `detect_reference_audio(video_path, reference_audio_path, extracted_audio_path=None, score_threshold=0.75)`

### 4.2 音频剥离

实现方式：

- 调用外部 `ffmpeg`
- 输出格式为 `pcm_s16le + 8000Hz + 单声道 wav`

默认输出路径：

- `recordings/analysis/<video_stem>.wav`

### 4.3 有声音判断

实现方式：

- 将音频按固定帧长切帧
- 逐帧计算 RMS
- 只要存在超过阈值的帧，就判定为“有声音”

返回：

- 是否有声音
- 活跃帧数量
- 最大 RMS
- 平均 RMS

### 4.4 随机音频包含判断

实现方式：

- 对录制音频和参考随机音频都计算帧 RMS 序列
- 对 RMS 序列做均值消除和能量归一化
- 使用滑窗点积计算最佳相似度
- 最佳分数超过阈值时，判定“包含参考随机音频”

优势：

- 不依赖复杂第三方 DSP 库
- 对随机音频这种高区分度样本比较合适

限制：

- 对强噪声、强压缩、严重失真场景可能需要调整阈值

## 5. 错误处理设计

异常类型：

- `HikvisionSDKError`
- `VideoAnalysisError`

原则：

- SDK 控制类错误由 SDK 层抛出
- 视频提取、音频解析、匹配失败由视频分析层抛出

## 6. 已知限制

- 当前项目仅面向 Windows 64 位环境
- 视频分析依赖本机安装 `ffmpeg`
- 随机音频匹配基于帧能量签名，不是完整声纹识别
- 音量控制路径对不同型号设备做了回退兼容，但个别型号可能仍需按真实报文微调

## 7. 变更维护要求

后续每次代码变更，默认同步执行以下动作：

1. 更新实现代码
2. 检查三份文档是否需要同步修改
3. 做最基本验证，例如 `python -m py_compile`
4. 使用详细 commit message 记录修改目标、行为变化和影响范围
5. 提交到当前分支
