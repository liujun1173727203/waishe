# 详细设计

## 1. 文件结构

### 1.1 `hikvision_voice.py`

SDK 封装层，负责：

- SDK 初始化与清理
- 设备登录与登出
- 获取当前音频编码配置
- 开启语音转发
- 开启实时流录像
- 保存录像文件

主要对象：

- `DeviceSession`
- `HikvisionVoiceSDK`
- `VoiceCall`
- `VoiceForwardSession`
- `StreamRecorder`

### 1.2 `hikvision_isapi.py`

ISAPI 封装层，负责：

- 统一发起 HTTP Digest 请求
- 查询双向对讲能力
- 查询和修正双向对讲输出配置
- 查询和修正复合流音频配置

主要对象：

- `CompositeStreamStatus`
- `AudioInputCapabilityStatus`
- `TwoWayAudioChannelStatus`
- `HikvisionIsapiClient`

### 1.3 `video_analysis.py`

录像分析层，负责：

- 从录像中提取音频
- 判断是否存在有效声音
- 判断是否匹配参考音频

### 1.4 `use_cases/voice_talk_cases.py`

语音发送用例层，负责：

- 生成三位数字参考音频
- 将 PCM 编码为设备支持格式
- 按帧切分编码音频
- 通过语音转发接口发送音频

主要对象：

- `PreparedRandomAudio`
- `RandomAudioTalkResult`
- `VoiceTalkUseCases`

### 1.5 `use_cases/speaker_test_cases.py`

扬声器测试用例编排层，负责：

- 检查设备是否支持扬声器输出
- 自动修正双向对讲配置
- 自动修正复合流带音频配置
- 录制并发送三位数字音频
- 对录像结果做声音与匹配分析

主要对象：

- `SpeakerTestResult`
- `SpeakerTestUseCases`

## 2. SDK 层设计

### 2.1 `HikvisionVoiceSDK`

主要接口：

- `initialize(...)`
- `login(...)`
- `logout(...)`
- `get_current_audio_compress(...)`
- `start_voice_forward(...)`
- `start_stream_record(...)`

### 2.2 `StreamRecorder`

执行流程：

1. 创建 `NET_DVR_PREVIEWINFO`
2. 调用 `NET_DVR_RealPlay_V40`
3. 调用 `NET_DVR_SaveRealData`
4. 在实时回调中累计接收字节数
5. 通过 `wait_for_first_data(...)` 等待首帧到达
6. 停止时依次调用 `NET_DVR_StopSaveRealData` 和 `NET_DVR_StopRealPlay`

## 3. ISAPI 层设计

### 3.1 请求方式

统一通过 `requests.request(...)` 发起请求：

- 协议：`http`
- 认证：`requests.auth.HTTPDigestAuth`
- 请求头：
  - `Accept: application/xml,text/xml,*/*`
  - `Content-Type: application/xml; charset=utf-8`，仅 `PUT` 时使用

### 3.2 双向对讲能力与配置

核心接口：

- `GET /ISAPI/System/TwoWayAudio/channels/capabilities`
- `GET /ISAPI/System/TwoWayAudio/channels/1`
- `PUT /ISAPI/System/TwoWayAudio/channels/1`

能力判断规则：

1. 读取 `audioOutputType` 节点
2. 检查其 `opt` 属性中是否包含 `Speaker`
3. 检查当前标签值是否为 `Speaker`
4. 读取 `microphoneVolume` 当前值和 `max` 属性
5. 判断 `microphoneVolume` 是否已经是最大值

自动修正规则：

1. 如果 `opt` 中不支持 `Speaker`，则判定设备不支持目标配置
2. 如果支持 `Speaker`，但当前值不是 `Speaker`，则需要下发配置
3. 如果 `microphoneVolume` 不是最大值，也需要下发配置
4. 下发报文时至少写入：
   - `id`
   - `enabled`
   - `audioOutputType`
   - `microphoneVolume`

### 3.3 复合流音频配置

核心接口：

- `GET /ISAPI/Streaming/channels/<trackStreamID>/capabilities`
- `GET /ISAPI/Streaming/channels/<trackStreamID>`
- `PUT /ISAPI/Streaming/channels/<trackStreamID>`

处理规则：

1. 先查询 `capabilities`
2. 判断是否存在 `Audio` 能力节点
3. 读取当前流配置
4. 查找 `Audio.enabled` 或 `Audio.enable`
5. 如果当前不是 `true`，则改为 `true` 后回写

## 4. 三位数字音频设计

### 4.1 参考音频生成

当前不再生成白噪声，而是随机生成三位数字，例如：

- `203`
- `581`
- `947`

每个数字使用双音频率合成，生成单声道 `8000Hz / pcm_s16le` 音频。

### 4.2 主要接口

- `prepare_random_audio_file(...)`
- `split_prepared_audio_frames(...)`
- `send_prepared_audio(...)`

输出结果包含：

- `digit_sequence`
- `pcm_bytes`
- `encoded_bytes`
- `frame_count`

### 5.3 用例执行具体步骤

以下步骤按当前代码真实执行顺序整理，包含调用的方法和接口名。

1. 校验输入参数  
   调用位置：`SpeakerTestUseCases.run_speaker_test(...)`  
   检查：
   - `record_duration_seconds > 0`
   - `send_duration_seconds > 0`
   - `send_duration_seconds <= record_duration_seconds`
   - `0 < similarity_threshold <= 1`

2. 检查设备是否支持音频输入  
   调用方法：`HikvisionIsapiClient.get_audio_input_capability_status(session)`  
   实际 ISAPI 接口：
   - `GET /ISAPI/System/TwoWayAudio/channels/capabilities`
   判断规则：
   - `audioOutputType` 的 `opt` 中必须包含 `Speaker`
   返回结果：
   - `AudioInputCapabilityStatus`

3. 计算测试输出目录和文件路径  
   生成：
   - 录像文件 `speaker_test_YYYYMMDD_HHMMSS.mp4`
   - 提取音频 `speaker_test_YYYYMMDD_HHMMSS.wav`
   - 参考音频 `random_audio_YYYYMMDD_HHMMSS.wav`

4. 检查并修正双向对讲输出配置  
   调用方法：`HikvisionIsapiClient.ensure_two_way_audio_speaker_ready(session)`  
   实际 ISAPI 接口：
   - `GET /ISAPI/System/TwoWayAudio/channels/capabilities`
   - `GET /ISAPI/System/TwoWayAudio/channels/1`
   - 必要时 `PUT /ISAPI/System/TwoWayAudio/channels/1`
   检查项：
   - `audioOutputType.opt` 是否支持 `Speaker`
   - `audioOutputType` 当前值是否为 `Speaker`
   - `microphoneVolume` 当前值是否等于 `max`
   修正项：
   - `enabled=true`
   - `audioOutputType=Speaker`
   - `microphoneVolume=max`

5. 检查并修正复合流音频配置  
   调用方法：`HikvisionIsapiClient.ensure_composite_stream_recording_enabled(...)`  
   实际 ISAPI 接口：
   - `GET /ISAPI/Streaming/channels/<trackStreamID>/capabilities`
   - `GET /ISAPI/Streaming/channels/<trackStreamID>`
   - 必要时 `PUT /ISAPI/Streaming/channels/<trackStreamID>`
   目的：
   - 确保录像流中带音频

6. 生成三位数字参考音频  
   调用方法：`VoiceTalkUseCases.prepare_random_audio_file(...)`  
   内部动作：
   - 读取设备音频编码：`HikvisionVoiceSDK.get_current_audio_compress(...)`
   - 随机生成三位数字
   - 合成三位数字音频
   - 按设备编码类型编码为 `G.711u` / `G.711a` / `PCM`
   输出：
   - `PreparedRandomAudio`
   - `digit_sequence`

7. 启动语音转发  
   调用方法：`HikvisionVoiceSDK.start_voice_forward(...)`  
   SDK 接口：
   - `NET_DVR_StartVoiceCom_MR_V30`

8. 启动实时录像  
   调用方法：`HikvisionVoiceSDK.start_stream_record(...)`  
   SDK 接口：
   - `NET_DVR_RealPlay_V40`
   - `NET_DVR_SaveRealData`

9. 等待录像流收到首帧  
   调用方法：`StreamRecorder.wait_for_first_data(...)`  
   校验目的：
   - 避免录像文件创建成功但内容为 `0` 字节

10. 按帧发送三位数字音频  
   调用方法：
   - `VoiceTalkUseCases.split_prepared_audio_frames(...)`
   - `VoiceForwardSession.send_encoded_audio(...)`
   SDK 接口：
   - `NET_DVR_VoiceComSendData`

11. 音频发送完成后继续录像到指定时长  
   逻辑：
   - 发送完音频帧后，继续等待剩余录像时长
   目的：
   - 保留设备实际播出的完整音频内容

12. 停止录像并校验文件大小  
   调用方法：
   - `StreamRecorder.stop()`
   - `SpeakerTestUseCases._wait_for_record_file(...)`
   SDK 接口：
   - `NET_DVR_StopSaveRealData`
   - `NET_DVR_StopRealPlay`
   校验：
   - 录像文件大小必须大于 `0`

13. 停止语音转发  
   调用方法：`VoiceForwardSession.stop()`  
   SDK 接口：
   - `NET_DVR_StopVoiceCom`

14. 从录像中提取音频并判断是否有声音  
   调用方法：`RecordedVideoAnalyzer.analyze_sound_presence(...)`  
   内部动作：
   - 提取录像音频
   - 转换为统一格式
   - 逐帧计算能量
   输出：
   - `SoundAnalysisResult`

15. 将录像音频与参考三位数字音做匹配  
   调用方法：`RecordedVideoAnalyzer.detect_reference_audio(...)`  
   输出：
   - `ReferenceAudioMatchResult`
   - `matched`
   - `best_score`
   - `threshold`

16. 返回结构化测试结果  
   返回对象：`SpeakerTestResult`

## 6. 输出目录规划

默认目录：

- `recordings/speaker_tests/<device-ip>/`

文件说明：

- `speaker_test_*.mp4`：录像文件
- `speaker_test_*.wav`：从录像中提取的音频
- `random_audio_*.wav`：三位数字参考音频

## 7. 错误处理

异常类型：

- `HikvisionSDKError`
- `VideoAnalysisError`
- `FileNotFoundError`
- `ValueError`

处理原则：

- SDK 调用失败时抛出 `HikvisionSDKError`
- ISAPI 请求失败时抛出 `HikvisionSDKError`
- 录像文件为空时抛出 `HikvisionSDKError`
- 音频分析失败时抛出 `VideoAnalysisError`
- 参数不合法时抛出 `ValueError`

## 8. 维护要求

后续每次代码变更，默认同步执行：

1. 更新实现代码
2. 检查并同步更新文档
3. 做基础验证，如 `python -m py_compile`
4. 使用明确的提交说明记录变更内容、影响范围和验证结果
