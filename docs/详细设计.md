# 详细设计

## 1. 文件结构

### 1.1 `hikvision_voice.py`

核心封装文件，包含：

- 常量定义
- HCNetSDK 结构体定义
- 语音和实时码流回调定义
- 数据类定义
- `HikvisionVoiceSDK` 主封装类
- `VoiceCall`、`VoiceForwardSession`、`StreamRecorder` 会话类

### 1.2 `demo_voice_call.py`

最小对讲示例。

### 1.3 `demo_stream_record.py`

最小实时取流录像示例。

### 1.4 `demo_composite_stream_record.py`

复合流录像示例，用于验证：

- 查询复合流能力
- 获取和修改流配置
- 开启音频使能
- 按设备 IP 分目录录像 10 秒

## 3. 结构体设计

### 3.1 `NET_DVR_LOCAL_SDK_PATH`

用途：

- 为 SDK 提供组件目录路径

### 3.2 `NET_DVR_LOCAL_TALK_MODE_CFG`

用途：

- 切换库模式对讲或 Windows API 对讲

### 3.3 `NET_DVR_COMPRESSION_AUDIO`

用途：

- 获取和设置设备语音编码参数

### 3.4 `NET_DVR_PREVIEWINFO`

用途：

- 向 `NET_DVR_RealPlay_V40` 传递预览通道、码流类型、链路方式等参数

### 3.5 `NET_DVR_XML_CONFIG_INPUT` / `NET_DVR_XML_CONFIG_OUTPUT`

用途：

- 通过 `NET_DVR_STDXMLConfig` 访问 ISAPI 配置接口

### 3.6 `NET_DVR_USER_LOGIN_INFO`

用途：

- 向 `NET_DVR_Login_V40` 传递登录参数

### 3.7 `NET_DVR_DEVICEINFO_V30` / `NET_DVR_DEVICEINFO_V40`

用途：

- 接收设备返回能力信息
- 提供默认对讲通道和默认预览通道

## 5. `HikvisionVoiceSDK` 设计

### 5.1 初始化

`initialize(enable_log=False, log_dir=None)`

处理内容：

- 加载 SDK
- 配置初始化路径
- 绑定接口签名
- 调用 SDK 初始化
- 按需开启日志

### 5.2 登录与登出

- `login(host, port, username, password)`
- `logout(session)`

### 5.3 对讲与语音转发

- `start_call(...)`
- `start_voice_forward(...)`

### 5.4 普通录像

`start_stream_record(session, file_path=None, channel=None, stream_type=0, link_mode=0, blocked=True, real_data_callback=None)`

设计说明：

- 创建 `StreamRecorder`
- 默认使用设备默认预览通道
- 默认使用主码流和 TCP 取流
- 通过 `NET_DVR_SaveRealData` 将码流直接保存到本地文件
- 未指定 `file_path` 时，自动按设备 IP 保存到 `recordings/streams/<device-ip>/`

### 5.5 ISAPI 透传与复合流切换

- `get_streaming_channel_capabilities(session, channel=None, stream_type=0)`
- `get_streaming_channel_config(session, channel=None, stream_type=0)`
- `set_streaming_channel_config(session, xml_body, channel=None, stream_type=0)`
- `ensure_composite_stream_recording_enabled(session, channel=None, stream_type=0)`

设计说明：

- 通过 `NET_DVR_STDXMLConfig` 访问 `/ISAPI/Streaming/channels/<trackStreamID>` 及其 capabilities 接口
- 通过 `channel` 和 `stream_type` 计算 `trackStreamID`
- 先判断能力节点中是否包含 `Audio`
- 如果当前不是复合流，则自动将 `./Audio/enable` 或 `./Audio/enabled` 设置为 `true`
- 配置成功后返回 `CompositeStreamStatus`

## 6. 会话对象设计

### 6.1 `VoiceCall`

职责：

- 启动和停止双向对讲
- 管理 `NET_DVR_StartVoiceCom_V30` 返回句柄

### 6.2 `VoiceForwardSession`

职责：

- 启动和停止编码音频转发
- 提供发送编码后音频能力

### 6.3 `StreamRecorder`

职责：

- 启动和停止实时预览
- 启动和停止录像文件保存
- 保存实时码流回调，防止被 Python 回收

关键方法：

- `start()`
- `stop()`
- `__enter__()`
- `__exit__()`

## 7. 回调设计

回调类型：

- `VOICE_DATA_CALLBACK`
- `REAL_DATA_CALLBACK`

适配逻辑：

- SDK 回调原始指针和长度
- Python 端通过 `ctypes.string_at()` 转换为 `bytes`
- 再交给上层 Python 回调处理

## 8. 错误处理设计

统一异常类型：

- `HikvisionSDKError`

错误来源：

- `NET_DVR_GetLastError`
- `NET_DVR_GetErrorMsg`
- `NET_DVR_STDXMLConfig`

## 9. 已知限制

- 当前项目仅面向 Windows 64 位环境
- 当前示例未包含自动化测试
- 录像文件保存格式取决于设备码流和 SDK 输出，扩展名由调用方按实际场景控制
- 部分设备返回的音频节点名称可能为 `enable` 或 `enabled`，当前实现两者都兼容

## 10. 变更维护要求

后续每次代码变更，默认同步执行以下动作：

1. 更新实现代码
2. 检查三份文档是否需要同步修改
3. 做最基本验证，例如 `python -m py_compile`
4. 使用详细 commit message 记录修改目标、行为变化和影响范围
5. 提交到当前分支
