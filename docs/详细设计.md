# 详细设计

- `DeviceSession`
- `HikvisionVoiceSDK`
- `VoiceCall`
- `VoiceForwardSession`
- `StreamRecorder`

### 1.2 `hikvision_isapi.py`

- XML 辅助函数
- `CompositeStreamStatus`
- `AudioVolumeStatus`
- `AudioIoVolumeStatus`
- `AudioInputCapabilityStatus`
- `HikvisionIsapiClient`

### 1.3 `video_analysis.py`

- `VideoAnalysisError`
- `SoundAnalysisResult`
- `ReferenceAudioMatchResult`
- `RecordedVideoAnalyzer`

### 1.4 `use_cases/voice_talk_cases.py`

随机音频能力模块，包含：

- `PreparedRandomAudio`
- `RandomAudioTalkResult`
- `VoiceTalkUseCases`

### 1.5 `use_cases/speaker_test_cases.py`

扬声器测试编排模块，包含：

- `SpeakerTestResult`
- `SpeakerTestUseCases`

## 2. SDK 层设计

### 2.1 `HikvisionVoiceSDK`

主要职责：

- SDK 初始化与清理
- 登录与登出
- 获取音频编码信息
- 开启语音转发
- 开启实时流录像
- 透传标准 ISAPI XML 配置

关键接口：

- `initialize(...)`
- `login(...)`
- `logout(...)`
- `start_voice_forward(...)`
- `start_stream_record(...)`
- `stdxml_config(...)`

### 2.2 `StreamRecorder`

实现流程：

1. 创建 `NET_DVR_PREVIEWINFO`
2. 调用 `NET_DVR_RealPlay_V40`
3. 调用 `NET_DVR_SaveRealData`
4. 停止时依次调用 `NET_DVR_StopSaveRealData` 和 `NET_DVR_StopRealPlay`

## 3. ISAPI 层设计

### 3.1 `HikvisionIsapiClient`

主要接口：

- `get_streaming_channel_capabilities(...)`
- `get_streaming_channel_config(...)`
- `set_streaming_channel_config(...)`
- `ensure_composite_stream_recording_enabled(...)`
- `set_audio_input_volume_to_max(...)`
- `set_audio_output_volume_to_max(...)`
- `ensure_audio_input_output_volume_max(...)`
- `get_audio_input_capability_status(...)`

### 3.2 音量调节策略

设计要点：

1. 输入和输出音量分别封装
2. 支持多组候选 ISAPI 路径回退
3. 优先读取 capabilities 中的 `max`
4. 若 capabilities 不可用，则尝试从配置节点推断最大值
5. 最后兜底使用默认值

### 3.3 复合流切换策略

设计要点：

1. 先读取 `/ISAPI/Streaming/channels/<trackStreamID>/capabilities`
2. 判断是否存在 `Audio` 能力节点
3. 再读取当前流配置
4. 兼容 `Audio.enable` 与 `Audio.enabled`
5. 若音频未启用，则将节点设置为 `true` 后重新下发

### 3.4 音频输入能力判断

设计要点：

1. 先尝试 `/ISAPI/System/Audio/capabilities`
2. 若失败，再尝试 `/ISAPI/System/TwoWayAudio/channels/1/capabilities`
3. 通过节点名或显式支持字段判断是否支持音频输入
4. 返回 `AudioInputCapabilityStatus` 供用例层直接消费

## 4. 随机音频模块设计

### 4.1 `PreparedRandomAudio`

字段：

- `file_path`

- `encode_type`
- `voice_channel`
- `pcm_bytes`
- `encoded_bytes`
- `frame_count`

用途：

- 表达“已生成、可直接发送”的参考音频数据

### 4.2 `VoiceTalkUseCases`

主要接口：

- `prepare_random_audio_file(...)`
- `send_prepared_audio(...)`
- `split_prepared_audio_frames(...)`
- `play_random_audio_file(...)`

设计说明：

- 将“生成音频”和“发送音频”拆开，便于复杂用例按步骤编排
- 保留 `play_random_audio_file(...)` 作为简单场景的一步式接口

## 5. 视频分析层设计

### 5.1 `RecordedVideoAnalyzer`

主要接口：

- `extract_audio(...)`
- `analyze_sound_presence(...)`
- `detect_reference_audio(...)`

### 5.2 声音存在性判断

实现方式：

1. 将音频统一转换为 `8000Hz / mono / pcm_s16le`
2. 按固定帧长切分
3. 逐帧计算 RMS
4. 统计超过阈值的有效帧

### 5.3 参考音频匹配

实现方式：

1. 计算录像与参考音频的帧 RMS 序列
2. 对序列做去均值和能量归一化
3. 通过滑窗点积寻找最佳匹配位置
4. 以相似度阈值判断是否命中

## 6. 扬声器测试用例设计

### 6.1 `SpeakerTestResult`

字段：

- `login_verified`
- `audio_input_status`
- `volume_status`
- `composite_status`
- `record_file_path`
- `reference_audio_path`
- `extracted_audio_path`
- `talk_result`
- `sound_result`
- `match_result`

### 6.2 `SpeakerTestUseCases.run_speaker_test(...)`

输入参数：

- `session`
- `record_channel`
- `voice_channel`
- `record_duration_seconds`
- `send_duration_seconds`
- `output_dir`
- `similarity_threshold`
- `amplitude_ratio`
- `seed`
- `per_frame_delay`

执行步骤：

1. 校验录制时长、发送时长和相似度阈值
2. 查询设备是否支持音频输入
3. 确定录像通道与对讲通道
4. 生成输出目录和目标文件路径
5. 设置音频输入、输出音量为最大
6. 确保复合流音频已开启
7. 生成随机参考音频并保存到设备 IP 对应目录
8. 开启对讲
9. 开启录像
10. 分帧发送随机音频
11. 录像持续到指定时长后停止
12. 剥离录像音频
13. 判断是否有声音
14. 判断是否匹配参考音频
15. 返回结构化结果

### 6.3 输出目录规划

默认目录：

- `recordings/speaker_tests/<device-ip>/`

文件：

- `speaker_test_*.mp4`：录像文件
- `speaker_test_*.wav`：从录像中剥离的音频
- `random_audio_*.wav`：原始参考音频

## 7. 错误处理

异常类型：

- `HikvisionSDKError`
- `VideoAnalysisError`
- `FileNotFoundError`
- `ValueError`

原则：

- 设备连接、对讲、录像、ISAPI 调用失败时抛出 `HikvisionSDKError`
- 音频提取和分析失败时抛出 `VideoAnalysisError`
- 用例参数非法时抛出 `ValueError`

## 8. 维护要求

1. 更新实现代码
2. 检查并同步更新三份文档
3. 做基础验证，如 `python -m py_compile`
4. 使用详细 commit message 记录变更目标、行为变化和验证结果
